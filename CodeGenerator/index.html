    代码生成器类结构展示

    晨曦 JAVA 类 设计
    创建时间:2020-04-02 22:11
    字数:592 阅读:
    1. 代码生成器结构如下图：
      1. 1.CXImport
      2. 2.EnumState
      3. 3.CXStruct
      4. 4.CXClass
      5. 5.CXField
      6. 6.CXFunction
      7. 7.CXInnerClass
      8. 8.CXSentence

    代码生成器结构如下图：

    1.CXImport

    ​ 表示一个完整类名，用来表示需要导入的包：java.lang.Object

    2.EnumState

    表示一个元素的状态，当状态冲突时抛出运行期异常：

    名称 介绍
    CLASS 表示这是一个类
    INTERFACE 表示这是一个接口
    ENUM 表示这是一个枚举类
    FINAL 表示该元素被final修饰
    ABSTRACT 表示该元素被abstract修饰
    PUBLIC 表示该元素被public修饰
    PRIVATE 表示该元素被private修饰
    PROTECTED 表示该元素被protected修饰

    3.CXStruct

    所有数据类的父类，被abstract修饰，其中可能包含的方法及元素如下：

    种类 类型 名称 介绍 是否为abstract
    元素 String name 存储标识符 否
    元素 Set<EnumState> states 存储状态量 否
    元素 Set<CXImport> imports 存储需要导入的包 否
    方法 public String toString() 获得字符串 是
    方法 public int hashCode() 获取hashCode() 是
    方法 public boolean equals(Object) 判断对象是否相等 是
    方法 public XXX getter() 否
    方法 public void setter(XXX) 否

    4.CXClass

    代表需要生成一个类，继承自CXStruct，其中可能包含的方法及元素如下：

    种类 类型 名称 介绍
    元素 Set<CXField> fields 存储类中包含的变量
    元素 Set<CXFunction> futcions 存储类中包含的函数
    元素 Set<CXInnerClass> innerClass 存储类中包含的内部类
    方法 public void read(File file) 从指定文件中读取离线的数据
    方法 public void write(File file) 向指定目录离线数据
    方法 public void readCode(StringBuilder) 读取需要转换为代码的数据
    方法 public XXX getter()
    方法 public void setter()

    5.CXField

    代表类中存储的一个元素，类似于这样的代码：public Object o

    种类 类型 名称 介绍
    元素 CXClass type 表示存储的元素类型
    方法 public XXX getter()
    方法 public void setter()

    6.CXFunction

    代表类中的一个方法：

    种类 类型 名称 介绍
    元素 CXType returnType 表示返回值的数据类型
    元素 Set<CXSentence> sentences 存储方法体内的语句
    方法 public boolean check(CXClass) 检查sentences内数据是否合法
    方法 public XXX getter()
    方法 public void setter()

    7.CXInnerClass

    代表内部类，继承自CXClass，其它实现与CXClass相同。

    8.CXSentence

    表示一个语句，如：int i = 0、for (int i = 0; i < 5; ++i)
